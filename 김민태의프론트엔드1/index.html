Chapter 01. 시작하며

  • 기술의 세가지 유형.

    • 1. 변하지 않는 기술: 네트워크, 운영체제, 컴퓨터시스템, 논리학, 대수학
    • 2. 느리게 변하는 기술: 프로그래밍 언어, 프로그래밍 패러다임, 자료구조, 보안, 알고리즘

    • 변하지 않는 기술일수록 높은 학습비용이 들고, 빠르게 변하는 기술일수록 낮은 학습비용이 든다.

    • 일관성
    • 유연성
    • 확장성
    • 독립성

Chapter 02. Javascript 그리고 Typescript

  • Javascript 변천사.

    • ES2015부터가 모던 Javascript이고, ES5.0(EcmaScript5.0) 버전이 주도적인 버전이라고 볼 수 있음. 매년 새로운 버전이 출시되지만 모든 안정성을 위해 변환기(트랜스 파일러)를 통해 5.0 버전으로 변환하기 때문.
    • Typescript도 일종의 트랜스 파일러 중 하나임. JavaScript을 기본적으로 지원하고 거기에 +@되기 때문에 기본적으로 JavaScript를 기본적으로 알고 있어야 활용하기 좋다.

  • 웹앱의 구성요소.

    • 웹앱을 실행시키는 역할을 하는 것이 브라우저. 브라우저가 실행되는 시간을 런타임이라고 하고 브라우저를 런타임 환경을 제공하는 환경이라고 이야기한다. 웹앱에게 런타임 환경을 제공하는 것은 브라우저가 될 수 있다.
    • 예전에는 브라우저만 런타임 환경을 제공했지만, node.js가 등장해서 브라우저가 아니어도 JavaScript를 실행시킬 수 있는 환경이 다수 만들어짐.
    • 웹서버라는 소프트웨어가 HTML 파일을 브라우저한테 전송 - 브라우저가 HTML을 해석해서 화면에 UI를 그림 - JavaScript 실행. JavaScript도 HTML을 조작할 수 있기 때문에 브라우저가 웹앱을 실행시킨 다음에도 UI가 Javascript에 의해 조작되는 경우도 있다.
    • 클라이언트 사이드 렌더링 : 실제로 브라우저에 최초 전송된 HTML에는 거의 내용이 없고, JavaScript가 실행되면서 필요한 UI를 그때그때마다 생성해 내는 방식
    • 서버 사이드 렌더링: 웹서버에서 HTML이 만들어져서 브라우저로 전송되는 방식
    • 웹앱에서는 HTML과 CSS로 그래픽 시스템을 표현할 수 있고, JavaScript로는 캔버스라는 태그로 2D, 3D 등의 그래픽을 표현할 수 있다. JavaScript로 그래픽을 표현할 수 있도록 만들어진 도구들을 API라고 함.
    • 그 외에도 미디어 파일을 다루거나 웹 워커, 웹 어셈블리 등 다양한 스펙들이 존재한다.
